# I2I Lookup Prompts: Product Standards × Idea to Implementation

Two prompt sets targeting the Idea to Implementation (I2I) process:

- **Part 1** (Prompts 1-39) — 39 Level 3 product standards checked against I2I artifacts
- **Part 2** (Prompts 40-47) — I2I phase prompts covering each stage of the I2I process

**Target site:** `[YOUR I2I SHAREPOINT SITE]` — replace this placeholder
with the actual SharePoint site name or URL before running each prompt.

**Purpose:** Determine where product management standards are already
embedded in (or conflict with) the Idea to Implementation process for
the largest initiatives, and where gaps exist. The I2I process governs
the biggest bets — alignment here has outsized impact.

---

## Response Format

Every prompt asks Copilot to return results in this structure:

```
## [Standard or Phase Name]

### Artifacts Found
| Artifact Name | Link | Last Updated | I2I Phase | What It Covers | Alignment |
|---|---|---|---|---|---|
| [Name] | [URL] | [Date] | [Phase] | [Brief description] | Aligns / Partially aligns / Different approach / Not addressed |

### What Differs
[For anything that partially aligns or takes a different approach — describe
what's different and recommend: adopt product standard, adopt I2I approach,
or synthesize (and say how)]

### Integration Opportunity
[Where could this standard be embedded into an existing I2I gate, template,
or checkpoint without creating a separate process?]

### Not Found
[If nothing found, say so explicitly and note what was searched]

### Confidence: [High / Medium / Low]
[Why]
```
