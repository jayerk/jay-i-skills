# SPM Lookup Prompts: Product Standards × Strategic Portfolio Management

41 prompts targeting the strategic portfolio management SharePoint site:

- **Part 1** (Prompts 1-39) — 39 Level 3 product standards checked against SPM artifacts
- **Part 2** (Prompt 40) — SPM assessment overlay: map SPM assessment criteria to product standards
- **Part 3** (Prompt 41) — SPM process inventory: what SPM already requires at each stage

**Target site:** `[YOUR SPM SHAREPOINT SITE]` — replace this placeholder
with the actual SharePoint site name or URL before running each prompt.

**Purpose:** Determine where product management standards are already
embedded in (or conflict with) strategic portfolio management processes,
and where gaps exist. SPM maturity is expected to be lower — many prompts
may return "Not Found." That's useful data: it tells you where product
standards fill a gap SPM hasn't addressed yet.

---

## Response Format

Every prompt asks Copilot to return results in this structure:

```
## [Standard Name]

### Artifacts Found
| Artifact Name | Link | Last Updated | SPM Process Area | What It Covers | Alignment |
|---|---|---|---|---|---|
| [Name] | [URL] | [Date] | [Area] | [Brief description] | Aligns / Partially aligns / Different approach / Not addressed |

### What Differs
[For anything that partially aligns or takes a different approach — describe
what's different and recommend: adopt product standard, adopt SPM approach,
or synthesize (and say how)]

### Integration Opportunity
[Where could this standard be embedded into an existing SPM process, template,
or checkpoint without creating a separate process?]

### Not Found
[If nothing found, say so explicitly and note what was searched]

### Confidence: [High / Medium / Low]
[Why — e.g., "Low — SPM site has limited content; searched document libraries
and pages but found few published standards"]
```
