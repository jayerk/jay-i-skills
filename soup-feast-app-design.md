# The Great Soup Feast — Product Requirements Document

> A standalone web app for organizing, voting on, and celebrating an annual soup competition. ~100 guests, ~25 soups, year 3 of the tradition.

---

## 1. Core Concepts

| Concept | Description |
|---|---|
| **Event** | A single Soup Feast (annual). One active event at a time, with an archive of past years. |
| **Soup** | A competition entry — name, cook, description, dietary tags, crockpot number. |
| **Guest** | An attendee. Identified by a unique link token — no account creation. |
| **Cook** | A guest who registered a soup. One soup per cook (or team). |
| **Vote** | A ranked-choice ballot. Each guest ranks their top 5 soups. |
| **Campaign** | A timed drip of messages (email + SMS via Twilio) leading up to the event. |

---

## 2. User Roles

| Role | Access |
|---|---|
| **Admin** | Everything — event setup, soup management, guest list, campaigns, budget, results reveal. Password-protected. |
| **Cook** | Sign up a soup, edit their entry, view leaderboard. Gets a unique cook link on registration. |
| **Guest** | RSVP, vote, view leaderboard, optionally leave tasting notes. Gets a unique guest link via invite. |

**No login walls.** Guests and cooks are identified by unique link tokens. Admin uses a simple password.

---

## 3. Decisions (Resolved)

These were open questions — now locked in:

| # | Decision | Resolution |
|---|---|---|
| 1 | **Guest identification** | Unique links per guest, auto-generated by the app. Admin just manages the guest list — no manual link wrangling. Postcard uses a shared QR code for RSVP; app handles individual tracking from there. |
| 2 | **Postcard design** | App exports event content + QR code. Jay designs the final postcard in Canva. |
| 3 | **Historical data** | Year 3. App includes an archive section. Years 1 & 2 can be backfilled with winner, runner-up, soup list, and any available photos. |
| 4 | **SMS provider** | Twilio. ~$5 total for 100 guests x 6 messages. $1.15/mo for the number. Worth it. |
| 5 | **Tasting notes privacy** | Default to private. Cooks receive a comprehensive anonymized summary (aggregate rating, flavor tag breakdown, all comments) at end of night. |
| 6 | **Leaderboard spoilers** | Show live top 5 during voting. Full results held for the admin-triggered reveal. |
| 7 | **Voting UX** | Guided tasting companion — not a wall of 25 soups. Card-based exploration with multiple browse modes (by table, by vibe, surprise me). Vote only after tasting 5+ soups. |
| 8 | **Post-vote flow** | After ballot submission, redirect to tasting notes page for soups the guest tried. Nudge notes for soups that need love. |
| 9 | **Save the Date timing** | September 2026. Sets the anchor early. App must be testable before this goes out. |

### 4.2 Soup Sign-Up & Idea Generator

**Sign-Up Flow:**
1. Cook lands on `/signup` (linked from invite or shared URL)
2. Enters: name, soup name, short description (2-3 sentences), dietary tags (multi-select: vegan, vegetarian, gluten-free, dairy-free, contains nuts, spicy)
3. Optional: "surprise entry" flag — hides soup name from public until reveal
4. Submits → receives confirmation page with their unique cook link (to edit later)
5. Admin sees the new entry in the soup roster immediately

**Idea Generator** (`/ideas` — "What Should I Make?"):

Two modes, available to anyone:

**Spin the Wheel:**
- Animated wheel with soup categories: Classic Comfort, Around the World, Vegan/Plant-Based, Weird & Wonderful, Crowd Pleasers
- Spin → lands on a category → shows a random soup idea from that category
- Tap to spin again
- Each idea shows: name, one-line description, dietary info, effort level

**Prompt Mode:**
- Answer 3 questions:
  1. Vegan or anything goes?
  2. Adventurous or crowd-pleaser?
  3. How much effort? (Crockpot & forget / Some love / Showstopper)
- Returns 3 tailored soup suggestions with descriptions
- "Not feeling these? Try again" button for 3 more

**Data source:** Static JSON file of ~100 curated soup ideas with tags (category, vegan, effort, description). No AI generation — just a well-curated list.

### 4.4 Day-of Experience: Guided Tasting + Voting

The app is a **tasting companion that ends in a vote** — not a ballot box with 25 items dumped on you.

**Entry:**
1. Guest scans QR code at the event OR opens their link from the SMS
2. Mobile-first — no app install, no login, no friction
3. Lands on the **Tasting Home** screen (not a voting page)

**Tasting Home — "How do you want to explore?"**

Three browse modes, always switchable:

| Mode | How It Works |
|---|---|
| **By Table** | Shows soups in crockpot number order — follows the natural walking path through the venue. Cards appear 3-4 at a time. "You're near Table 2 — here's what's there." |
| **By Vibe** | Filter by what sounds good: "Comfort," "Adventurous," "Light & Fresh," "Spicy," "Vegan/Plant-Based." Shows matching soups as a focused list. |
| **Surprise Me** | One soup card at a time, random order. Swipe to see the next one. Low-commitment discovery. |

**Soup Cards:**
- Big, tappable, phone-optimized
- Shows: crockpot number, soup name, cook name, short description, dietary tags
- Two actions per card:
  - **"Tried it"** — marks the soup as tasted. Optionally opens a quick tasting note (1-5 ladles + flavor tags + 140-char note). Tasting note is skippable.
  - **"Skip for now"** — moves on, keeps the soup in the untasted queue

**Tasting Progress:**
- Persistent progress bar at top: "You've tried 7 of 25 soups"
- Subtle encouragement at milestones: "Halfway there!" at 12-13

**Discovery Nudges (the recommendation engine):**

Context-aware suggestions that appear naturally during the tasting journey:

| Trigger | Nudge |
|---|---|
| Guest has tried 5-8 soups | "You've been going heavy on comfort soups — Soup #19 (West African Peanut Stew) is a crowd favorite you haven't hit yet." |
| Dietary pattern match | "You liked 3 vegan soups — Soup #22 is the only vegan entry you haven't tried." |
| Popularity gap | "Soup #4 only has 6 votes so far — could be a hidden gem worth tasting." |
| Table proximity (if table info available) | "You're near Table 3 — Soups #9 and #11 are right there." |
| Guest hasn't tried a top-performing soup | "Soup #14 is in the current top 5 and you haven't tasted it yet." |

Nudges are non-intrusive — they appear as banner cards between soup cards, not as popups. Dismissible. Max 1 nudge per 3 soups viewed.

**Implementation notes for nudges:**
- Nudge logic runs client-side using the guest's tasted/untasted list + live vote counts from the leaderboard API
- "By table" proximity nudges use the crockpot number ordering — no GPS needed
- Dietary pattern matching compares the guest's tasted soups' dietary tags to untasted soups
- Popularity data comes from the same real-time feed as the leaderboard

**Vote Gate — "Ready to vote?"**

- Sticky button appears at the bottom after the guest has marked **5+ soups as tried**
- Before 5 soups: button is visible but grayed out — "Try at least 5 soups before voting"
- Tapping opens the **Ranking Screen**:
  - Shows ONLY the soups the guest has tried (not all 25)
  - Pre-sorted by the guest's tasting note ratings (if they left any — highest-rated first)
  - Guest drags to reorder OR taps to add to their top 5 ranking
  - Clear "Your Top 5" section (sticky footer)
  - Can still browse and add more soups — "Tried another one?" link returns to tasting mode
- Submit ballot → fun confirmation: "Your taste buds have spoken!"
- One ballot per unique token. Can update until admin closes voting.

**Post-Vote Redirect:**

After ballot submission, the experience doesn't dead-end:

1. **Confirmation screen** with animation and their ranking summary
2. **"Leave tasting notes?"** CTA — links to a focused tasting notes page showing:
   - Their ranked soups first (with any existing notes pre-populated)
   - Then other soups they tried but didn't rank
   - Quick-tap interface: ladles + flavor tags + optional text
3. **"These soups could use some love"** section at the bottom — highlights soups with the fewest tasting notes so far. Encourages coverage.
4. **Leaderboard link** — "See how the race is shaping up" → `/leaderboard`

This flow converts the dead-end "thanks for voting" moment into continued engagement.

**Ranked Choice Voting (RCV) — Instant Runoff:**
- Round 1: Count all #1 votes. If any soup has >50% of first-place votes → winner.
- Round 2+: Eliminate the soup with the fewest first-place votes. Redistribute those ballots to each voter's next-ranked remaining soup. Repeat.
- Tiebreaker for elimination: if two soups tie for fewest votes, eliminate the one with fewer second-place votes. If still tied, eliminate the one registered later (arbitrary but deterministic).

**Computed Awards:**
- **Champion**: RCV winner
- **Runner-Up**: Last soup eliminated (2nd place in RCV)
- **Sleeper Hit**: Biggest positive jump — soup that gained the most vote share from Round 1 to its final round
- **Most Polarizing**: Highest standard deviation in ranking position across all ballots
- **Crowd Comfort**: Most combined "hearty" + "savory" tasting note tags (if tasting notes active)
- **Boldest Bowl**: Most combined "spicy" + "smoky" + "umami" tags (if tasting notes active)

### 4.7 Results Reveal

Hidden until admin triggers it. This is the ceremony.

**Admin Trigger:**
- Admin goes to `/admin/reveal` → clicks "Reveal Results"
- Results page (`/results`) goes live for everyone
- All guest links now show a "Results are in!" banner linking to `/results`
- Day-after campaign message auto-sends (if scheduled)

**Reveal Page Flow:**

1. **Sankey Diagram** — the centerpiece
   - Animated visualization of RCV vote flow
   - Left: all 25 soups with Round 1 vote counts
   - Each elimination round shows votes redistributing to next-ranked soups
   - Eliminated soups fade out, surviving soups grow
   - Right: winner highlighted with celebration animation
   - Color-coded per soup — consistent colors throughout

2. **Final Rankings** — Full 1st through 25th ordered list

3. **Awards** — Champion, Runner-Up, Sleeper Hit, Most Polarizing, Crowd Comfort, Boldest Bowl (last two only if tasting notes were used)

4. **Soup Stories** (if tasting notes active) — Per-soup card: aggregate ladle rating, top flavor tags, highlighted guest notes

5. **Fun Stats Recap:**
   - Total votes cast
   - Average soups ranked per guest (did people rank all 5?)
   - Most-ranked soup (appeared on the most ballots)
   - Hidden Gem (fewest ballots — people missed it)
   - Voting time distribution (early birds vs. last-minute voters)
   - First vote timestamp, last vote timestamp

6. **"Your Ballot vs. Results"** (`/results/my/:token`)
   - Personal comparison: "You ranked X first — they finished 3rd!"
   - Shareable result card (generated image) for social media
   - Variations: "I voted for the winner!", "My #1 was the Sleeper Hit!", "I found the Hidden Gem!"

## 9. RCV Algorithm

```
function runRCV(ballots, soups):
    rounds = []
    remaining = [...soups]

    while remaining.length > 1:
        // Count first-place votes among remaining soups
        counts = {}
        for each remaining soup: counts[soup] = 0

        for ballot in ballots:
            topPick = first item in ballot.rankings that is in remaining
            if topPick: counts[topPick] += 1

        totalVotes = sum of all counts
        round = { counts: {...counts}, eliminated: null, redistributed: {} }

        // Check for majority winner
        for soup, count in counts:
            if count > totalVotes / 2:
                rounds.push(round)
                return { winner: soup, rounds, finalRankings: buildRankings(rounds) }

        // Find soup to eliminate (fewest first-place votes)
        minCount = min of counts values
        candidates = soups where counts[soup] == minCount

        // Tiebreaker: fewest second-place votes, then latest registration
        eliminated = breakTie(candidates, ballots, remaining)

        // Track where eliminated soup's votes go
        redistributed = {}
        for ballot in ballots:
            if first remaining pick == eliminated:
                nextPick = second remaining pick in ballot.rankings
                if nextPick: redistributed[nextPick] = (redistributed[nextPick] || 0) + 1

        round.eliminated = eliminated
        round.redistributed = redistributed
        remaining.remove(eliminated)
        rounds.push(round)

    return { winner: remaining[0], rounds, finalRankings: buildRankings(rounds) }
```

The `redistributed` map in each round is what powers the Sankey diagram — it shows exactly where each eliminated soup's votes went.

## 11. Non-Functional Requirements

| Requirement | Target |
|---|---|
| **Mobile-first** | Voting, leaderboard, and results pages must be fully usable on a phone. Desktop is secondary. |
| **No login for guests** | Zero friction. Token-based links only. |
| **Performance** | Voting page loads in <2s on 4G. Leaderboard updates within 3s of a vote. |
| **Offline resilience** | If a guest loses signal mid-vote, the ballot should be cached locally and submitted when connection returns. |
| **Capacity** | 100 concurrent voters. Not a scale challenge — but the real-time leaderboard needs to handle rapid updates gracefully. |
| **Data privacy** | No personal data beyond name + contact info. Tasting notes anonymized for cook summaries. Guest data deleted 90 days post-event (or kept for archive with consent). |
| **Accessibility** | Voting page should be usable with screen readers. Color is not the only indicator in the leaderboard. |
