# Copilot Research Prompts: Internal Artifact Inventory

39 prompts — one per Level 3 (Established) standard across 10 sub-practices.
Run each prompt in M365 Copilot to search Confluence, SharePoint, and Teams
for existing artifacts that already support the standard.

**Purpose:** Inventory what exists before creating anything new.

**How to use the results:** Paste each response into the summary table at the
bottom of this file. The completed table becomes a heat map of coverage x
confidence across all 39 standards.

---

## Response Format

Every prompt asks Copilot to return results in this structure:

```
Search Confluence, SharePoint, Teams wikis, and linked internal docs for
any standards, playbooks, or process guidance related to the standard below.
Return only substantive artifacts — skip anything that mentions the topic
in passing.

## Standard to Validate
[Paste your standard here]

## Output Format

### Artifacts Found
| Artifact Name | Link | Source System | Last Updated | What It Covers | Alignment |
|---|---|---|---|---|---|
| [Name] | [URL] | [e.g., Confluence / SharePoint] | [Date] | [1-line summary] | Aligns / Partial / Different |

### What Differs
For each Partial or Different artifact:
- What specifically differs
- Recommendation: adopt ours / adopt theirs / synthesize
- If synthesize: one-sentence proposed resolution

### Conflicts
[Flag any cases where two existing artifacts contradict each other]

### Not Found
[If nothing relevant, state what systems and topic areas were searched]

### Confidence: [High / Medium / Low]
[One line — what was searched, what couldn't be accessed]
```
